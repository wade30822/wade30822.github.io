Spring Boot

最后还想说说Spring Boot，这是我最近接触到的一个新框架，感觉这个框架用处很大，所以在这里说说。Spring boot是Spring官方的一个子项目，按照官方的说法，Spring Boot能够让你快速开发部署用于生产环境下的Spring项目，它遵循约定大于配置的原则。关于约定大于配置的原则，我在这里多说几句，到底什么是约定大于配置，简单的来说就是这个框架有一个规范，比如说配置文件放在哪个文件夹下，取什么名字，配置文件的格式规范是什么样子，这叫做约定。当然这个约定是可以不遵守的，你完全可以自定义，这叫做配置。约定大于配置的意思就是希望开发者按照它的规范来使用这个框架，而不是自定义配置，这个原则在很多框架上都是被强调的。这么做有两点好处：

  1. 加快框架初始化和执行速度；框架在初始化时需要根据配置文件的内容对需要用到的类进行初始化，那么它将首先在规范中规定的配置文件位置去寻找配置文件，如果按照规范来，那么框架就一下子找到了配置文件，如果按照自定义配置来的话，框架先得遍历规范中的位置然后再到配置的文件路径去寻找，这无疑拖慢了速度。
  2. 减少开发者的错误几率；一个框架往往有很多东西需要配置，如果开发者自己定义这些配置方式难免会犯一些错误，有些错误可能会影响整个框架的运行。如果根据规范来做的话，犯错可能性会降低许多。

那么Spring Boot跟这篇文章有什么关系呢？答案是Spring Boot提供了内置的tomcat服务器程序，不仅如此，还包括了像Jetty，Undertow这些很流行的服务器，这使得我们可以无需手动部署war到服务器上。还有一点也是Spring Boot带给我们的便利，那就是我们可以不用XML配置，直接使用java代码或注解写出一个纯java的应用。当然，要想使用Spring Boot的话需要一些配置，这里以maven为例：

有人可能会问不是说好的纯java应用吗，怎么又多了个pom.xml,这里要注意一下，pom.xml是maven的配置文件，maven是一个构建工具，它可以构建一个应用，上面所说得纯java应用指的就是这个构建的应用。

然后，在项目中创建一个启动类，代码如下：

@SpringBootApplication
public class Example {
    public static void main(String[] args) throws Exception {
        SpringApplication.run(Example.class, args);
    }

这串代码需要注意两点，第一点就是@SpringBootApplication这个注解，之所以可以构建一个纯java应用而不需要XML配置文件，就是因为这个注解，具体原理不是本文重点，第二个要注意的地方就是倒数第二行的代码，这就是整个应用的入口，不管是java本地应用还是web应用，都是这句代码。这样web应用的开发就变得和开发java本地应用一样了。最后我们将执行mvn spring-boot:run命令，对于web应用，这条命令会编译，打包wen程序，然后将其部署到它内置的tomcat服务器上运行，具体过程如下所示：
